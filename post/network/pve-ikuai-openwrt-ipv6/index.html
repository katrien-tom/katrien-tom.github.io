PVE-IKuai-openwrt-旁路由

PVE 安装设置

1. 下载安装镜像并安装

下载地址1：https://www.proxmox.com/en/downloads/proxmox-virtual-environment/iso/proxmox-ve-7-4-iso-installer

下载地址2：http://download.proxmox.com/iso/

使用rufus制作启动盘,下载地址：https://rufus.ie/zh/

2. 安装完成设置

安装完成后进入https://ip地址:8006进入PVE后台地址

2.1 设置PVE直通特性功能

Intel CPU

shell里面输入命令：

vim /etc/default/grub

在里面找到：

GRUB_CMDLINE_LINUX_DEFAULT="quiet"

然后修改为

GRUB_CMDLINE_LINUX_DEFAULT="quiet intel_iommu=on"

在更新一下

update-grub

AMD CPU

shell里面输入命令：

vim /etc/default/grub

在里面找到：

GRUB_CMDLINE_LINUX_DEFAULT="quiet"

然后修改为

GRUB_CMDLINE_LINUX_DEFAULT="quiet amd_iommu=on"

在更新一下

update-grub

新增所需模块

修改文件/etc/modules,加入如下的行

vim /etc/modules

在里面填入

vfio
vfio_iommu_type1
vfio_pci
vfio_virqfd

执行命令来更新initramfs

update-initramfs -u -k all

重启PVE虚拟机

2.2 直通硬盘设置

查看设备序列号：ls -l /dev/disk/by-id/

选择直通的硬盘(sata2根据磁盘多少来设置数字)：qm set 101 -sata2 /dev/disk/by-id/ata-ST1000121212-1234_1234

IKuai 安装设置

1. 设置ikuai后台地址

输入2 进入设置WAN/LAN地址，然后再输入0 设置LAN1地址，设置ikuai后台地址,默认账户/密码：admin/admin

2. 进入ikuai设置网络信息

  • ikuai主界面，点击wan口,绑定对应网卡

  • 网络设置->内外网设置->lan口进入->高级设置->扩展网卡->将网卡绑定

  • 网络设置->DHCP设置->DHCP服务器

    • 192.168.66.29-192.168.66.99，网关填写iKuai地址（192.168.66.253）

    • 192.168.66.100-192.168.66.199，网关填写旁路由（192.168.66.252）

    • 想哪个设备走旁路由，就将此设备设置为静态ip（192.168.66.100 - 192.168.66.199之间）

3. IPV6设置

OpenWrt

1. 编译固件

  • 进入网址https://firmware-selector.immortalwrt.org后选择系统架构和版本

  • 预安装软件列表添加

    luci-i18n-firewall-zh-cn luci-i18n-filebrowser-zh-cn luci-app-argon-config luci-i18n-argon-config-zh-cn luci-i18n-opkg-zh-cn luci-i18n-ttyd-zh-cn luci-i18n-passwall-zh-cn luci-app-openclash luci-i18n-homeproxy-zh-cn openssh-sftp-server

    精简版

    luci-i18n-firewall-zh-cn luci-app-argon-config luci-i18n-argon-config-zh-cn luci-i18n-opkg-zh-cn luci-i18n-ttyd-zh-cn luci-i18n-passwall-zh-cn openssh-sftp-server

    如果不知道软件名称 可以在仓库搜索和查询，点击下方直达仓库搜索https://mirrors.vsean.net/openwrt/releases/23.05.4/packages/x86_64/luci/?sort=size&order=desc

  • 首次启动时运行的脚本（uci-defaults）

    # 设置默认wan口防火墙打开 方便虚拟机用户首次访问webui
uci set firewall.@zone[1].input='ACCEPT'
uci commit firewall
# 设置主机名映射 解决安卓原生TV首次连不上网的问题
uci add dhcp domain
uci set "dhcp.@domain[-1].name=time.android.com"
uci set "dhcp.@domain[-1].ip=203.107.6.88"
uci commit dhcp
# 根据网卡数量配置网络
count=0
for iface in $(ls /sys/class/net | grep -v lo); do
  # 检查是否有对应的设备，并且排除无线网卡
  if [ -e /sys/class/net/$iface/device ] && [[ $iface == eth* || $iface == en* ]]; then
    count=$((count + 1))
  fi
done
if [ "$count" -eq 1 ]; then
    # 单个网卡，设置为 DHCP 模式
    uci set network.lan.proto='dhcp'
    uci commit network
elif [ "$count" -gt 1 ]; then
    # 多个网卡，保持静态 IP
    uci set network.lan.ipaddr='192.168.100.1'
    uci commit network
fi

  • 点击请求构建

  • 选择COMBINED-EFI镜像下载

2. 固件img扩容

前提条件安装parted软件

apt-get install parted -y

如果想增加2GB空间 则可用如下方法设置，遇到提示就按照要求输入Fix 和 OK

# 解压缩镜像文件
gzip -kd immortalwrt.img.gz
# 给img镜像末尾填充2GB空数据 (count=2000 表示增加 2GB 的空间)
dd if=/dev/zero bs=1M count=2000 >> immortalwrt.img

# 为了将2GB空间真正赋予具体的分区，使用分区工具parted操作镜像。
parted immortalwrt.img

# 查看分区情况（有提示"Error: The backup GPT table is corrupt, but the primary appears OK, so that will be used"输入OK回车）
print
# 输入Fix

# 调整分区大小 (将第 2 个分区扩展至镜像文件的 100%)
resizepart 2 100%
# 查看是否扩展成功2412MB
print
# 退出分区工具
quit

sed -e 's,https://downloads.immortalwrt.org,https://mirrors.cernet.edu.cn/immortalwrt,g' \
    -e 's,https://mirrors.vsean.net/openwrt,https://mirrors.cernet.edu.cn/immortalwrt,g' \

4. 安装docker

确定软件包是有足够的空间的，如果你安装的是我扩容之后的固件，自然没问题。docker和相关组件可能占用1g

luci-i18n-dockerman-zh-cn

上述名称是docker的名称。我们可以在软件包那里先更新列表，然后搜索该组件，并安装即可。比较大，需要好几分钟。

另外一个方法是采用命令行安装。

opkg update
opkg install luci-i18n-dockerman-zh-cn

5. PVE 安装Openwrt

正常创建虚拟机

  • 操作系统选择->不使用任何存储介质
  • CPU->2核
  • 内存2G
  • 机型：Host

后续设置

  • 将磁盘分离并删除
  • 上传openwrt镜像，完成后复制上传路径或者在PVE下方任务找到"复制数据"，找到openwrt的上传路径并复制
# 转换openwrt的img文件为虚拟磁盘
qm importdisk 102 /var/lib/vz/template/iso/openwrt.img local-lvm
# 以上代码102对应的为虚拟机， /var/lib/vz/template/iso/openwrt.img对应的是上传的路径
  • 回到PVE中openwrt有个未使用磁盘->总线设置为SATA
  • 引导顺序调整到第一位并勾选上
  • 开机更改IP地址为192.168.66.252，网关地址配置为IKuai地址192.168.66.253
