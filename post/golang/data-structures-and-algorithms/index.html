1. 二分查找

1.1 二分查找

二分查找的原理

  1. 前提条件： 数组必须是已排序的。
  2. 基本思想：
    • 从数组的中间元素开始判断。
    • 如果目标值等于中间元素，查找成功。
    • 如果目标值小于中间元素，则可以确定目标值在中间元素的左侧，接着在左侧继续查找。
    • 如果目标值大于中间元素，则目标值在中间元素的右侧，接着在右侧继续查找。
  3. 时间复杂度： 每次都将搜索范围缩小一半，所以时间复杂度是 O(log n)，比线性查找的 O(n) 要高效得多。

二分查找的步骤

  1. 初始化左右指针： 左指针 left 设置为数组的起始位置（0），右指针 right 设置为数组的结束位置（length - 1）。
  2. 查找循环：
    • 计算中间位置：mid = (left + right) / 2（避免溢出的写法是 mid = left + (right - left) / 2）。
    • 比较目标值和中间值：
      • 如果目标值等于中间值，查找成功，返回 mid。
      • 如果目标值小于中间值，更新 right = mid - 1。
      • 如果目标值大于中间值，更新 left = mid + 1。
  3. 循环结束条件： 当 left > right 时，说明数组中没有目标值，查找失败。

二分查找的代码实现

package main

import "fmt"

// 二分查找函数
func binarySearch(arr []int, target int) int {
    left, right := 0, len(arr)-1

    for left <= right {
        mid := left + (right-left)/2 // 防止溢出

        if arr[mid] == target {
            return mid // 找到目标值，返回下标
        } else if arr[mid] < target {
            left = mid + 1 // 目标值在右侧
        } else {
            right = mid - 1 // 目标值在左侧
        }
    }
    return -1 // 没有找到目标值，返回 -1
}

func main() {
    arr := []int{1, 3, 5, 7, 9, 11, 13, 15}
    target := 7

    result := binarySearch(arr, target)
    if result != -1 {
        fmt.Printf("目标值 %d 在数组中的下标是: %d\n", target, result)
    } else {
        fmt.Println("目标值没有在数组中找到")
    }
}

关键点总结：

  1. 必须排序： 二分查找只能在已排序的数组中进行。
  2. 查找过程高效： 每次都将搜索范围缩小一半，因此时间复杂度是 O(log n)。
  3. 循环控制： 使用 left <= right 来确保循环继续，直到找到目标或搜索范围为空。

1.2 二分区间查找

1.2.1 基本思想

与常规的二分查找不同，二分区间查找不是寻找一个单一的目标值，而是查找某个区间范围，或者在某个条件下找到适合的区间。

举例说明

假设你有一个有序数组，数组中每个元素代表一个区间的左边界，你需要找出一个给定的值应该落在这个数组中的哪个区间内。

1.2.2 应用场景

  • 查找某个值落在哪个区间。
  • 查找某个值小于等于或大于等于某个区间。
  • 例如：给定一个区间数组 [0, 5, 10, 20, 30]，你需要查找数字 12 落在哪个区间（即，10 <= 12 < 20）。

1.2.3 查找的步骤

步骤1：初始化区间

  • 对于一个有序数组 arr，首先定义两个指针 left 和 right，分别指向数组的起始位置和末尾位置。

步骤2：计算中间区间

  • 计算中间位置：mid = left + (right - left) / 2，这样可以避免溢出。
  • 根据中间值判断：
    • 如果目标值落在 arr[mid] 这个区间内，则找到目标区间。
    • 如果目标值小于 arr[mid] 的右边界，则继续向左查找。
    • 如果目标值大于 arr[mid] 的右边界，则向右查找。

步骤3：更新指针

  • 根据比较结果更新 left 或 right，逐步缩小查找范围。

步骤4：终止条件

  • 如果 left > right，说明查找范围已经没有合适的区间，可以结束查找。

1.2.4. 代码示例

​ 我们假设有一个区间数组 intervals，每个元素是一个包含左右边界的区间，例如 [[0, 5], [10, 20], [30, 40]]。给定一个值 target，我们要找到目标值应该落在哪个区间内。

package main

import "fmt"

// 区间查找函数
func binaryIntervalSearch(intervals [][2]int, target int) (int, bool) {
    left, right := 0, len(intervals)-1
    
    // 二分查找
    for left <= right {
        mid := left + (right - left) / 2
        start, end := intervals[mid][0], intervals[mid][1]
        
        if target >= start && target < end {
            return mid, true // 目标值在该区间内
        } else if target < start {
            right = mid - 1 // 目标值可能在左边的区间
        } else {
            left = mid + 1 // 目标值可能在右边的区间
        }
    }
    
    return -1, false // 没有找到目标区间
}

func main() {
    // 有序区间数组
    intervals := [][2]int{
        {0, 5},
        {10, 20},
        {30, 40},
        {50, 60},
    }

    target := 12
    index, found := binaryIntervalSearch(intervals, target)

    if found {
        fmt.Printf("目标值 %d 落在区间 %v 内，区间的下标是: %d\n", target, intervals[index], index)
    } else {
        fmt.Println("目标值没有落在任何区间内")
    }
}

1.2.5 代码解析

  1. 输入：
    • 输入是一个有序的区间数组 intervals，每个区间由两个数表示左边界和右边界。
    • 目标值 target 需要找到在哪个区间内。
  2. 查找过程：
    • 利用二分查找的思想，逐步缩小查找范围。
    • 每次根据中间的区间与目标值的比较来决定查找方向。
    • 如果 target 落在 interval[mid] 区间内，返回区间的索引。
  3. 终止条件：
    • 如果找到目标区间，返回该区间的索引。
    • 如果查找范围为空，说明目标值不在任何区间内。

1.2.6 时间复杂度

  • 时间复杂度：与标准二分查找一样，O(log n)，因为每次都会将查找范围缩小一半。
  • 空间复杂度：O(1)，只用了常数的额外空间。

1.2.7 扩展应用

  • 区间匹配： 对于给定的多个目标值，利用二分区间查找可以高效地找到每个值在哪个区间内。
  • 区间合并： 有时候你可能需要合并重叠的区间，二分查找能帮助你快速定位需要合并的区间。
  • 范围查询： 对于查询某个值是否属于指定区间的范围，二分查找提供了一种非常高效的方式。

1.2.8 总结

  • 二分区间查找 是一种扩展了标准二分查找的算法，它用于查找一个值在区间数组中的位置。
  • 它广泛应用于那些需要根据给定值快速找到合适区间的问题，时间复杂度为 O(log n)，非常高效。
  • 这种方法的核心思想仍然是 逐步缩小查找范围，但是在具体实现上需要根据区间的边界进行特殊处理。

2. 链表

2.1 双向链表

2.1.1 双向链表的结构

  • 节点结构： 每个节点包含三个部分：
    • 数据部分（data）： 存储数据元素。
    • 下一个节点指针（next）： 指向链表中的下一个节点。
    • 前一个节点指针（prev）： 指向链表中的前一个节点。
  • 链表结构：
    • 头节点（head）： 指向链表的第一个节点。
    • 尾节点（tail）： 指向链表的最后一个节点。
    • 头尾指针： 双向链表通过头尾指针提供了对链表两端的快速访问。

2.1.2 双向链表的特点

  • 双向访问： 可以从链表的任意一个节点向前和向后遍历。
  • 内存消耗： 每个节点需要存储两个指针（next 和 prev），相比于单向链表，内存消耗较大。
  • 插入和删除： 双向链表支持在任意位置进行插入和删除操作，并且可以在常数时间内完成（假设我们已经定位到要插入或删除的节点）。

2.1.3 双向链表的常见操作

插入操作

  • 在头部插入节点：
    • 新建一个节点，将其 next 指针指向原头节点，将原头节点的 prev 指针指向新节点。
    • 然后更新头指针为新节点。
  • 在尾部插入节点：
    • 新建一个节点，将其 prev 指针指向原尾节点，将原尾节点的 next 指针指向新节点。
    • 更新尾指针为新节点。
  • 在中间插入节点：
    • 需要找到要插入的位置（假设我们已经定位到前一个节点），将新节点的 prev 指针指向前一个节点，next 指针指向后一个节点。
    • 修改前一个节点的 next 指针和后一个节点的 prev 指针，分别指向新节点。

删除操作

  • 删除头节点：
    • 更新头指针为原头节点的下一个节点，并将新头节点的 prev 指针设置为 null。
  • 删除尾节点：
    • 更新尾指针为原尾节点的前一个节点，并将新尾节点的 next 指针设置为 null。
  • 删除中间节点：
    • 找到要删除的节点，将该节点的前后节点的指针相互指向（前一个节点的 next 指向后一个节点，后一个节点的 prev 指向前一个节点）。
    • 然后释放该节点。

遍历操作

  • 从头到尾遍历： 从头节点开始，按照 next 指针访问每个节点。
  • 从尾到头遍历： 从尾节点开始，按照 prev 指针访问每个节点。

查找操作

  • 查找操作通常是线性查找，即从头部或尾部开始遍历链表，查找目标值。

2.1.4 双向链表的代码实现

以下是双向链表的简单实现，包含了插入、删除和遍历操作。

package main

import "fmt"

// 定义链表节点结构
type Node struct {
    data int   // 节点存储的值
    next *Node // 指向下一个节点的指针
    prev *Node // 指向前一个节点的指针
}

// 定义双向链表结构
type DoublyLinkedList struct {
    head *Node // 链表的头节点
    tail *Node // 链表的尾节点
}

// 在链表头插入节点
func (list *DoublyLinkedList) InsertAtHead(data int) {
    newNode := &Node{data: data}
    if list.head == nil {
        list.head = newNode
        list.tail = newNode
    } else {
        newNode.next = list.head
        list.head.prev = newNode
        list.head = newNode
    }
}

// 在链表尾插入节点
func (list *DoublyLinkedList) InsertAtTail(data int) {
    newNode := &Node{data: data}
    if list.tail == nil {
        list.head = newNode
        list.tail = newNode
    } else {
        newNode.prev = list.tail
        list.tail.next = newNode
        list.tail = newNode
    }
}

// 删除头节点
func (list *DoublyLinkedList) DeleteAtHead() {
    if list.head == nil {
        fmt.Println("链表为空")
        return
    }
    if list.head == list.tail {
        list.head = nil
        list.tail = nil
    } else {
        list.head = list.head.next
        list.head.prev = nil
    }
}

// 删除尾节点
func (list *DoublyLinkedList) DeleteAtTail() {
    if list.tail == nil {
        fmt.Println("链表为空")
        return
    }
    if list.head == list.tail {
        list.head = nil
        list.tail = nil
    } else {
        list.tail = list.tail.prev
        list.tail.next = nil
    }
}

// 从头到尾遍历链表
func (list *DoublyLinkedList) TraverseFromHead() {
    current := list.head
    for current != nil {
        fmt.Print(current.data, " ")
        current = current.next
    }
    fmt.Println()
}

// 从尾到头遍历链表
func (list *DoublyLinkedList) TraverseFromTail() {
    current := list.tail
    for current != nil {
        fmt.Print(current.data, " ")
        current = current.prev
    }
    fmt.Println()
}

func main() {
    // 创建一个空的双向链表
    list := &DoublyLinkedList{}

    // 在链表头插入节点
    list.InsertAtHead(10)
    list.InsertAtHead(20)
    list.InsertAtHead(30)

    // 在链表尾插入节点
    list.InsertAtTail(40)
    list.InsertAtTail(50)

    fmt.Println("从头到尾遍历链表：")
    list.TraverseFromHead() // 输出: 30 20 10 40 50

    fmt.Println("从尾到头遍历链表：")
    list.TraverseFromTail() // 输出: 50 40 10 20 30

    // 删除头节点
    list.DeleteAtHead()
    fmt.Println("删除头节点后：")
    list.TraverseFromHead() // 输出: 20 10 40 50

    // 删除尾节点
    list.DeleteAtTail()
    fmt.Println("删除尾节点后：")
    list.TraverseFromHead() // 输出: 20 10 40
}

2.1.5 代码解析

  1. 节点定义：
    • Node 结构体表示链表的每个节点，包含 data（存储的数据），next（指向下一个节点的指针），和 prev（指向前一个节点的指针）。
  2. 链表定义：
    • DoublyLinkedList 结构体表示双向链表，包含 head 和 tail，分别指向链表的头节点和尾节点。
  3. 操作：
    • 插入节点： 提供了在头部和尾部插入节点的方法。
    • 删除节点： 提供了删除头节点和尾节点的方法。
    • 遍历链表： 提供了从头到尾和从尾到头的遍历方法。

2.1.6 时间复杂度分析

  • 插入操作： 插入操作（无论是头部还是尾部）均为 O(1) 时间复杂度。
  • 删除操作： 删除头节点或尾节点也是 O(1) 时间复杂度。
  • 查找操作： 查找元素时，必须遍历链表，时间复杂度为 O(n)，其中 n 是链表的长度。
  • 空间复杂度： 由于每个节点需要存储两个指针，空间复杂度为 O(n)。

2.1.7 优缺点总结

优点：

  • 双向链表支持从头到尾和从尾到头的遍历，灵活性更强。
  • 插入和删除操作比数组更高效（O(1)），特别是当我们知道要插入或删除的节点位置时。
  • 在某些应用中，双向链表比单向链表更加高效，例如需要频繁进行前后插入删除的情况。

缺点：

  • 每个节点需要额外的空间来存储前一个节点的指针。
  • 相比单向链表，双向链表的内存消耗较大。
  • 操作稍复杂，插入和删除时需要更新前后节点的指针。

2.2 跳表SkipList

​ 跳表（Skip List）是一种基于多层有序链表的随机化数据结构，用来在 O(log n) 时间内完成查找、插入和删除操作。它的核心思想是通过在链表上建立多级“索引”来加快查找速度，有点像高速公路和普通道路的关系。

2.2.1 跳表的设计初衷

普通的有序单链表查找元素需要从头遍历，时间复杂度是 O(n)。 跳表通过在链表上加“跳跃层”（索引层），让查找速度接近 O(log n)，相当于把链表的线性查找改造成类似二分查找的效率。

2.2.2 跳表的结构

跳表是由多层链表组成的，底层是完整的有序链表，每一层都是下层的“索引”，层数越高节点越少。

示意图（数字代表节点值，箭头代表指针）：

Level 3:  1 -------------------------> 17 ---------------------> 31
Level 2:  1 ---------> 9 -------------> 17 ---------> 25 -------> 31
Level 1:  1 -> 5 -> 9 -> 13 -> 17 -> 21 -> 25 -> 29 -> 31 -> 37

特点：

  • 最底层包含所有节点。
  • 每一层都是下层的一个子集（通过概率随机决定谁升上去）。
  • 每个节点可能有多个层级的前进指针（向右）和下移指针（向下）。

2.2.3 跳表的查找过程

假设要在上图中查找 25：

  1. 从最高层的第一个节点开始（1）。
  2. 在当前层一直往右走，直到下一个节点的值大于目标值（或为空）。
  3. 向下一层移动，继续往右走。
  4. 重复这个过程，直到到底层。
  5. 如果底层的节点值等于目标值，则查找成功。

查找时间复杂度：

  • 高度约为 log₂n（因为每层节点数约是下层的一半）。
  • 每层最多走几个节点，所以总复杂度是 O(log n)。

2.2.4 跳表的插入过程

插入一个值 x：

  1. 从最高层开始，找到每一层中应该插入的位置（记录下插入点）。
  2. 在底层插入节点。
  3. 以 50% 的概率将该节点“晋升”到上一层，并在上一层相应位置插入。
  4. 重复晋升步骤，直到随机失败或到达最高层。

插入时间复杂度：O(log n) （因为查找插入点是 O(log n)，晋升层数是随机的，期望值也是 O(log n)）

2.2.5 跳表的删除过程

删除一个值 x：

  1. 从最高层开始查找 x。
  2. 在每一层中找到 x 节点并删除（调整指针）。
  3. 向下一层继续删除对应节点。
  4. 直到最底层完成删除。

删除时间复杂度：O(log n)

2.2.6 跳表的关键参数

  • 最大层数（Max Level）： 一般设为 log₂n 或一个固定常数（如 16）。
  • 晋升概率（p）： 常用 0.5，表示节点有 50% 的概率晋升到上一层。
  • 空间复杂度： O(n)，但会有额外的索引节点存储。

2.2.7 跳表与其他数据结构的比较

2.2.8 跳表（简化版）

package main

import (
    "fmt"
    "math/rand"
    "time"
)

const (
    MAX_LEVEL = 16
    P         = 0.5
)

type Node struct {
    value int
    next  []*Node
}

type SkipList struct {
    head  *Node
    level int
}

func NewNode(value, level int) *Node {
    return &Node{
        value: value,
        next:  make([]*Node, level),
    }
}

func NewSkipList() *SkipList {
    return &SkipList{
        head:  NewNode(0, MAX_LEVEL),
        level: 1,
    }
}

func randomLevel() int {
    level := 1
    rand.Seed(time.Now().UnixNano())
    for rand.Float64() < P && level < MAX_LEVEL {
        level++
    }
    return level
}

func (sl *SkipList) Search(target int) *Node {
    current := sl.head
    for i := sl.level - 1; i >= 0; i-- {
        for current.next[i] != nil && current.next[i].value < target {
            current = current.next[i]
        }
    }
    current = current.next[0]
    if current != nil && current.value == target {
        return current
    }
    return nil
}

func (sl *SkipList) Insert(value int) {
    update := make([]*Node, MAX_LEVEL)
    current := sl.head
    for i := sl.level - 1; i >= 0; i-- {
        for current.next[i] != nil && current.next[i].value < value {
            current = current.next[i]
        }
        update[i] = current
    }

    level := randomLevel()
    if level > sl.level {
        for i := sl.level; i < level; i++ {
            update[i] = sl.head
        }
        sl.level = level
    }

    newNode := NewNode(value, level)
    for i := 0; i < level; i++ {
        newNode.next[i] = update[i].next[i]
        update[i].next[i] = newNode
    }
}

func (sl *SkipList) Delete(value int) {
    update := make([]*Node, MAX_LEVEL)
    current := sl.head
    for i := sl.level - 1; i >= 0; i-- {
        for current.next[i] != nil && current.next[i].value < value {
            current = current.next[i]
        }
        update[i] = current
    }

    current = current.next[0]
    if current != nil && current.value == value {
        for i := 0; i < sl.level; i++ {
            if update[i].next[i] != current {
                break
            }
            update[i].next[i] = current.next[i]
        }
        for sl.level > 1 && sl.head.next[sl.level-1] == nil {
            sl.level--
        }
    }
}

3.Jaccard相似度

Jaccard 相似度是用来衡量两个集合相似度的一种指标， 它的值介于 0 和 1 之间：

  • 1 表示两个集合完全相同
  • 0 表示两个集合没有任何交集

它特别适合用来比较“集合型数据”的相似性，比如关键词集合、标签集合、用户兴趣集合等。

应用场景

  1. 文本相似度计算
    • 把文本分成词（或 n-gram）后，作为集合比较。
    • 例如用来检测抄袭、相似新闻聚类。
  2. 推荐系统
    • 比较用户的兴趣集合相似度，推荐相似用户喜欢的内容。
  3. 数据去重
    • 比较数据项的特征集合，如果相似度大于一定阈值，就认为是重复项。
  4. 图像/视频检索
    • 对图片生成“特征集合”，用 Jaccard 计算相似度。

Jaccard 距离

有时需要“差异度”而不是“相似度”，可以用 Jaccard 距离：

package main

import (
	"cmp"
	"fmt"
)

func JaccardTimeConsuming[T comparable](collection1, collection2 []T) float64 {
	if len(collection1) == 0 || len(collection2) == 0 {
		return 0.0
	}
	map1 := make(map[T]struct{}, len(collection1))
	for _, ele := range collection1 {
		map1[ele] = struct{}{}
	}
	map2 := make(map[T]struct{}, len(collection2))
	for _, ele := range collection2 {
		map2[ele] = struct{}{}
	}
	intersection := 0 //交集的个数
	for key := range map1 {
		if _, exists := map2[key]; exists {
			intersection += 1
		}
	}
	return float64(intersection) / float64(len(collection1)+len(collection2)-intersection)
}
func JaccardForSorted[T cmp.Ordered](collection1, collection2 []T) float64 {
	if len(collection1) == 0 || len(collection2) == 0 {
		return 0.0
	}
	intersection := 0 // 交集的个数
	for i, j := 0, 0; i < len(collection1) && j < len(collection2); {
		if collection1[i] == collection2[j] {
			intersection += 1
			i += 1
			j += 1
		} else if collection1[i] < collection2[j] {
			i += 1
		} else {
			j += 1
		}
	}
	return float64(intersection) / float64(len(collection1)+len(collection2)-intersection)
}

func main() {
	collection1 := []int{1, 2, 3, 4, 5}
	collection2 := []int{3, 4, 5, 6, 7}
	fmt.Println(JaccardTimeConsuming(collection1, collection2)) // 0.42857142857142855
	fmt.Println(JaccardForSorted(collection1, collection2))     // 0.42857142857142855
}

4. 快速排序

4.1 快速排序

4.1.1 快速排序是什么

快速排序（Quick Sort）是一种 分治法（Divide and Conquer） 的排序算法，通过选择一个“基准值”（pivot），把数组分成两部分，然后递归地排序这两部分。 它是平均性能非常高的排序算法之一，平均时间复杂度是 O(n log n)。

4.1.2 核心思想

  1. 选基准（pivot）：从数组中挑一个值（常见是第一个、最后一个或随机选择）。
  2. 分区（partition）：把比基准小的放左边，比基准大的放右边。
  3. 递归：分别对左边和右边再进行快速排序。
  4. 合并：因为分区过程已经是原地交换，合并时不需要额外操作。

4.1.3 排序过程示例

假设我们要排序：

[5, 2, 9, 1, 5, 6]

步骤：

  1. 选基准 pivot = 5（这里选第一个元素）。
  2. 分区：
    • 左边：小于 5 → [2, 1]
    • 右边：大于等于 5 → [9, 5, 6]
  3. 递归排序：
    • 排 [2, 1] → [1, 2]
    • 排 [9, 5, 6]（pivot=9） → [5, 6, 9]
  4. 合并结果：
[1, 2, 5, 5, 6, 9]

4.1.4 代码实现

package main

import (
    "fmt"
)

func quickSort(arr []int) {
    if len(arr) < 2 {
        return
    }
    left, right := 0, len(arr)-1

    pivotIndex := len(arr) / 2 // 选中间值作为基准
    arr[pivotIndex], arr[right] = arr[right], arr[pivotIndex]

    for i := range arr {
        if arr[i] < arr[right] {
            arr[left], arr[i] = arr[i], arr[left]
            left++
        }
    }

    arr[left], arr[right] = arr[right], arr[left]

    quickSort(arr[:left])
    quickSort(arr[left+1:])
}

func main() {
    arr := []int{5, 2, 9, 1, 5, 6}
    quickSort(arr)
    fmt.Println(arr)
}

4.2 用分治寻找topK

4.2.1 问题定义

我们有一个长度为 n 的数组 arr，要找出其中前 K 个最大元素（或最小元素）。 要求比直接排序 O(n log n) 更快，尤其是当 K 远小于 n 时。

4.2.2 分治思想的切入点

这个问题可以用 快速选择算法（Quickselect）解决，它就是分治的一个变形版本：

  1. 选一个枢轴（pivot）
  2. 把数组分为两部分：
    • 大于 pivot 的一边
    • 小于 pivot 的另一边
  3. 判断 K 在哪一边：
    • 如果左边的元素数 ≥ K → 递归左边
    • 如果左边的元素数 < K → 递归右边（注意要调整 K）

这样平均时间复杂度可以做到 O(n)，最坏情况是 O(n²)（pivot 选得很糟时）。

4.2.3 核心算法步骤（Top K 最大值）

假设我们要找前 K 个最大值：

func topK(nums []int, k int) []int {
    if k <= 0 || k > len(nums) {
        return nil
    }
    quickSelect(nums, 0, len(nums)-1, k)
    return nums[:k] // 前K个即结果（未必有序）
}

func quickSelect(nums []int, left, right, k int) {
    if left >= right {
        return
    }

    pivotIndex := partition(nums, left, right)

    leftSize := pivotIndex - left + 1 // 左边(含pivot)元素个数
    if k < leftSize {
        quickSelect(nums, left, pivotIndex-1, k)
    } else if k > leftSize {
        quickSelect(nums, pivotIndex+1, right, k-leftSize)
    }
}

// partition 按 pivot 从大到小划分
func partition(nums []int, left, right int) int {
    pivot := nums[left]
    l, r := left, right

    for l < r {
        for l < r && nums[r] <= pivot {
            r--
        }
        nums[l] = nums[r]
        for l < r && nums[l] >= pivot {
            l++
        }
        nums[r] = nums[l]
    }
    nums[l] = pivot
    return l
}

k := 2

  • 平均情况：O(n) 因为每次分区只递归一边，分治树平均高度是常数级。
  • 最坏情况：O(n²) 如果每次 pivot 都是最大/最小值（比如数组已排序）。
  • 优化点：
    • 用随机 pivot 避免最坏情况（随机化 Quickselect）。
    • 当 n 很大而 K 很小时，性能优势明显。

5. 堆

5.1 堆

5.1.1 什么是堆（Heap）

堆是一种 完全二叉树（Complete Binary Tree） 结构，分为两种类型：

  • 大顶堆（Max Heap）：每个节点的值 ≥ 子节点的值（堆顶是最大值）
  • 小顶堆（Min Heap）：每个节点的值 ≤ 子节点的值（堆顶是最小值）

堆常用于：

  • 优先队列（Priority Queue）
  • Top K 问题
  • 堆排序

5.1.2 堆的结构特点

  1. 完全二叉树 除了最后一层，其他层都是满的，最后一层从左到右依次填充。
  2. 堆序性 大顶堆：父节点 ≥ 子节点 小顶堆：父节点 ≤ 子节点
  3. 存储方式 通常用 数组 存储，而不是指针结构，因为完全二叉树用数组能高效定位父子节点：
    • 父节点下标：parent = (i - 1) / 2
    • 左子节点下标：left = 2 * i + 1
    • 右子节点下标：right = 2 * i + 2

5.1.3 常用操作

5.1.3.1 插入（Insert）
  1. 把新元素放到数组最后。
  2. 上浮（Heapify Up）：比较新元素和父节点，如果违反堆序，交换，直到满足条件。
5.1.3.2 删除堆顶（Pop / Extract）
  1. 取出堆顶（最大值或最小值）。
  2. 把最后一个元素移到堆顶。
  3. 下沉（Heapify Down）：比较堆顶与子节点，交换到正确位置。
5.1.3.3 建堆（Build Heap）
  • 方法一：插入法（O(n log n)）
  • 方法二：原地建堆（从最后一个非叶子节点开始下沉，O(n)）

5.1.4 Go 语言示例（大顶堆）

package main

import (
    "fmt"
)

type MaxHeap struct {
    data []int
}

// 上浮
func (h *MaxHeap) heapifyUp(index int) {
    for index > 0 {
        parent := (index - 1) / 2
        if h.data[index] > h.data[parent] {
            h.data[index], h.data[parent] = h.data[parent], h.data[index]
            index = parent
        } else {
            break
        }
    }
}

// 下沉
func (h *MaxHeap) heapifyDown(index int) {
    lastIndex := len(h.data) - 1
    for {
        left := index*2 + 1
        right := index*2 + 2
        largest := index

        if left <= lastIndex && h.data[left] > h.data[largest] {
            largest = left
        }
        if right <= lastIndex && h.data[right] > h.data[largest] {
            largest = right
        }

        if largest != index {
            h.data[index], h.data[largest] = h.data[largest], h.data[index]
            index = largest
        } else {
            break
        }
    }
}

// 插入
func (h *MaxHeap) Push(val int) {
    h.data = append(h.data, val)
    h.heapifyUp(len(h.data) - 1)
}

// 弹出堆顶
func (h *MaxHeap) Pop() int {
    if len(h.data) == 0 {
        panic("heap is empty")
    }
    root := h.data[0]
    last := h.data[len(h.data)-1]
    h.data = h.data[:len(h.data)-1]
    if len(h.data) > 0 {
        h.data[0] = last
        h.heapifyDown(0)
    }
    return root
}

func main() {
    h := &MaxHeap{}
    nums := []int{3, 10, 5, 6, 2, 8, 15}
    for _, v := range nums {
        h.Push(v)
    }
    fmt.Println("Heap:", h.data)
    fmt.Println("Pop:", h.Pop())
    fmt.Println("Heap after Pop:", h.data)
}

  4. 重复直到排序完成。

5.2 用堆寻找topK

在 Go 语言的 container/heap 包中，其实没有固定的小顶堆或大顶堆概念。container/heap 提供的是一个通用的堆接口，它依赖你实现的 Less 方法来决定堆的顺序。

具体来说：

type Interface interface {
    sort.Interface  // 包含 Len(), Less(i, j int), Swap(i, j int)
    Push(x any)
    Pop() any
}
  • 如果你在 Less(i, j int) 中返回 h[i] < h[j]，那么堆就是小顶堆（根节点最小）。
  • 如果你返回 h[i] > h[j]，那么堆就是大顶堆（根节点最大）。

所以 container/heap 是可定制的，可以实现小顶堆或大顶堆，取决于你 Less 方法的实现。

思路

在 Go 中，可以使用 container/heap 包实现堆来维护 TopK 元素：

  1. 维护一个长度不超过 K 的小顶堆

    • TopK 最大值 → 小顶堆

    • TopK 最小值 → 大顶堆

  2. 遍历数组：

    • 当堆的长度小于 K → 直接 push。
    • 当前元素比堆顶更“极端” → 弹出堆顶，再 push 当前元素。

  3. 遍历结束后，堆中的元素就是 TopK（无序）。

  4. 若需要有序输出，可再对结果切片排序。

时间复杂度

  • 插入堆：O(logK)
  • 遍历 N 个元素：总复杂度 O(N logK)
  • 当 K 远小于 N 时，效率优于直接排序 O(N logN)。

TopK 最大值（小顶堆）

package main

import (
	"container/heap"
	"fmt"
)

// 小顶堆
type MinHeap []int

func (h MinHeap) Len() int           { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h MinHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *MinHeap) Push(x any)        { *h = append(*h, x.(int)) }
func (h *MinHeap) Pop() any {
	old := *h
	n := len(old)
	x := old[n-1]
	*h = old[:n-1]
	return x
}

func topKMax(nums []int, k int) []int {
	if k <= 0 {
		return nil
	}
	h := &MinHeap{}
	heap.Init(h)

	for _, num := range nums {
		if h.Len() < k {
			heap.Push(h, num)
		} else if num > (*h)[0] {
			heap.Pop(h)
			heap.Push(h, num)
		}
	}

	res := make([]int, h.Len())
	for i := 0; h.Len() > 0; i++ {
		res[i] = heap.Pop(h).(int)
	}
	return res
}

func main() {
	nums := []int{3, 1, 5, 12, 2, 11}
	k := 3
	fmt.Println("TopK 最大值:", topKMax(nums, k))
}

6. LRU缓存

6.1. LRU缓存概述

  • **LRU缓存(Least Recently Used，最近最少使用）**缓是一种缓存淘汰策略，最近最少使用的元素会被移除。它通常用于内存缓存中，以提升程序性能。
  • 在缓存满时，LRU会删除长时间未访问的条目，保证缓存中保存的是最常用的数据。

6.2. LRU缓存的工作原理

  • 缓存大小有限，当新的元素被插入时，如果缓存已满，则会删除最久未使用的元素（尾部元素）。
  • 每次访问缓存时，数据会被标记为最近使用（移到头部），减少重复数据的存储。

6.3. Go语言LRU缓存的基本实现

典型的Go语言LRU缓存实现会使用哈希表和双向链表。哈希表用来存储缓存的数据，双向链表用来记录数据的访问顺序。

  • 哈希表：提供O(1)的查询、插入时间复杂度。
  • 双向链表：在O(1)时间复杂度内，可以移动节点到链表的头部（表示最近使用），或者从尾部删除节点（表示最久未使用）。

6.4. 关键数据结构

  • 双向链表节点：每个节点存储一个键值对，表示缓存中的数据。双向链表的结构允许在常数时间内删除或移动节点。
  • 哈希表：通过哈希表的键（key）可以快速定位到双向链表中的对应节点。
  • LRU缓存类：该类包含一个固定容量的缓存，当缓存满时，移除最久未访问的数据。

6.5 Go语言实现LRU缓存的核心代码结构

package algorithm

import "container/list"

// LRU缓存，不支持并发。
type LRUCache struct {
	cache map[int]string // 存储业务数据
	lst   list.List      // 辅助的链表，用于决定淘汰谁
	cap   int            // 缓存容量的上限
}

func NewLRUCache(capacity int) *LRUCache {
	lru := new(LRUCache)
	lru.cache = make(map[int]string)
	lru.lst = list.List{}
	lru.cap = capacity
	return lru
}

// 向缓存中添加元素（添加之前先通过调Get确保key不在缓存里）。时间复杂度0（1）
func (lru *LRUCache) Add(key int, value string) {
	if len(lru.cache) == lru.cap { // 刚刚到达缓存容量上限
		// 先从缓存中淘汰一个元素
		back := lru.lst.Back()
		delete(lru.cache, back.Value.(int)) // interface {} is nil,not int
		lru.lst.Remove(back)
	}
	// 将新元素添加到缓存中
	lru.cache[key] = value
	lru.lst.PushFront(key)
}

// 从辅助链表中找一个元素。时间复杂度0（N)
func (lru *LRUCache) find(key int) *list.Element {
	if lru.lst.Len() == 0 {
		return nil
	}
	head := lru.lst.Front()
	for {
		if head == nil {
			break
		}
		if head.Value.(int) == key {
			return head
		} else {
			head = head.Next()
		}
	}
	return nil
}

// 从缓存中查找元素。时间复杂度0（N）
func (lru *LRUCache) Get(key int) (string, bool) {
	value, exists := lru.cache[key]
	ele := lru.find(key)
	if ele != nil {
		lru.lst.MoveToFront(ele)
	}
	return value, exists
}

7. 超时缓存

7.1 超时缓存概述

  • 超时缓存（TTL Cache，Time To Live Cache）：在缓存中存储的数据设置有效期（过期时间）。
  • 核心机制：超过设定的过期时间后，数据会被视为失效（无效数据）。
  • 区别于LRU：
    • LRU 关注的是“使用频率”，淘汰最久未访问的数据。
    • 超时缓存关注的是“时间”，过了指定时间即失效，即使很频繁访问也会过期。
  • 常用于 临时存储、防止缓存雪崩 或 自动清理过时数据。

7.2 超时缓存的工作原理

  • 每个缓存项除了存储 key-value 外，还包含一个 过期时间（expireAt）。
  • 写入缓存：
    • 记录数据和过期时间。
  • 读取缓存：
    • 如果当前时间 > 过期时间，缓存数据视为无效，删除并返回未命中。
  • 定期清理：
    • 通过后台 goroutine 定期扫描缓存，清理已过期的条目，避免内存泄漏。

7.3 超时缓存的实现

package algorithm

import (
	"container/heap"
	"time"
)

type HeapNode struct {
	value    int // 对应到map里的key
	deadline int //到期时间戳，精确到秒
}
type TimeoutHeap []*HeapNode

func (heap TimeoutHeap) Len() int {
	return len(heap)
}
func (heap TimeoutHeap) Less(i, j int) bool {
	return heap[i].deadline < heap[j].deadline // 小顶堆
}

func (heap TimeoutHeap) Swap(i, j int) {
	heap[i], heap[j] = heap[j], heap[i]
}
func (heap *TimeoutHeap) Push(x any) {
	node := x.(*HeapNode)
	*heap = append(*heap, node)
}
func (heap *TimeoutHeap) Pop() (x any) {
	n := len(*heap)
	last := (*heap)[n-1]
	// 删除最后一个元素
	*heap = (*heap)[0 : n-1]
	return last //返回最后一个元素
}

// 超时缓存，不支持并发
type TimeoutCache struct {
	cache map[int]any // 存储业务数据
	hp    TimeoutHeap // 辅助堆，用于决定淘汰谁
	cap   int         // 缓存容量的上限
}

func NewTimeoutCache(cap int) *TimeoutCache {
	tc := new(TimeoutCache)
	tc.cache = make(map[int]any, cap)
	tc.hp = make(TimeoutHeap, 0, 10)
	tc.cap = cap
	heap.Init(&tc.hp) // 包装升级，从常规的slice升级为堆
	go tc.taotai()
	return tc
}

// 向缓存中添加元素。时间复杂度未到容量上限：O（1）至到达容量上限O(logN)
func (tc *TimeoutCache) Add(key int, value any, life int) {
	// 计算新元素的deadline
	deadline := int(time.Now().Unix()) + life
	if len(tc.cache) == tc.cap { //刚刚到达缓存容量上限
		top := tc.hp[0]
		if top.deadline <= deadline { //堆顶元素比新元素先到期。注意这里是<=，意味着在过期时间
			// 淘汰堆顶
			heap.Pop(&tc.hp)
			delete(tc.cache, top.value)
		} else { // 新元素比堆顶元素先到期，则新元素不能放入缓存
			return
		}
	}
	// 直接把key value放入map
	tc.cache[key] = value
	// 把key和deadline放入堆
	node := &HeapNode{value: key, deadline: deadline}
	heap.Push(&tc.hp, node)
}

// 从缓存中查找元素。时间复杂度0（1)
func (tc TimeoutCache) Get(key int) (any, bool) {
	value, exists := tc.cache[key]
	return value, exists
}

// 不停地检查堆中是否有到期的元素，将其删除
func (tc *TimeoutCache) taotai() {
	// 按deadline构建的小根堆，所以堆顶元素是最早到期的。每次for循环只需要检查堆顶就可以了，因为堆顶被删除了
	for {
		if tc.hp.Len() == 0 {
			time.Sleep(100 * time.Millisecond)
			continue
		}
		now := int(time.Now().Unix())
		top := tc.hp[0]
		if top.deadline < now { // 堆顶的到期时间比当前时刻早，说明堆顶到期了
			heap.Pop(&tc.hp)
			delete(tc.cache, top.value)
		} else { // 堆顶还没有到期
			time.Sleep(100 * time.Millisecond) //稍作休息，减少cPU开销
		}
	}
}

8. 归并排序**（Merge Sort）**

8.1 基本概念

  • 归并排序（Merge Sort） 是一种 分治算法：
    1. 分：把数组一分为二，递归地排序左右子数组。
    2. 治：合并两个有序子数组，形成一个整体有序数组。
  • 特点：
    • 稳定排序（相同元素不会交换相对位置）
    • 时间复杂度：O(n log n)
    • 空间复杂度：O(n)（需要额外数组存放合并结果）

8.2 思路拆解

  1. 递归拆分数组
    • 不断把数组拆成左右两半，直到子数组长度 ≤ 1（自然有序）。
  2. 合并两个有序数组
    • 使用双指针 i, j 遍历两个子数组，比较当前元素大小，较小者放入结果数组。
    • 剩下未放完的部分直接拼接。

8.3 Go 实现

package main

import "fmt"

// 合并两个有序数组
func merge(left, right []int) []int {
    result := make([]int, 0, len(left)+len(right))
    i, j := 0, 0

    for i < len(left) && j < len(right) {
        if left[i] <= right[j] {
            result = append(result, left[i])
            i++
        } else {
            result = append(result, right[j])
            j++
        }
    }

    // 把剩余元素拼接上去
    result = append(result, left[i:]...)
    result = append(result, right[j:]...)
    return result
}

// 归并排序
func mergeSort(arr []int) []int {
    if len(arr) <= 1 {
        return arr
    }
    mid := len(arr) / 2
    left := mergeSort(arr[:mid])
    right := mergeSort(arr[mid:])
    return merge(left, right)
}

func main() {
    arr := []int{38, 27, 43, 3, 9, 82, 10}
    sorted := mergeSort(arr)
    fmt.Println("排序结果:", sorted)
}

8.4 执行流程示例

假设输入：

[38, 27, 43, 3]

Step 1: 拆分

[38, 27] 和 [43, 3]

Step 2: 继续拆

[38] [27]  |  [43] [3]

Step 3: 合并

[27, 38]   |   [3, 43]

Step 4: 再合并

[3, 27, 38, 43]

最终得到有序数组。

8.5 复杂度分析

  • 时间复杂度
    • 每层拆分需要 O(n) 次合并
    • 拆分深度 log n
    • 总复杂度：O(n log n)
  • 空间复杂度
    • 递归调用 + 临时数组存储合并结果
    • O(n)

8.6 总结

  • 归并排序利用 分治思想，先分后治。
  • 优点：稳定，适合处理链表/大数据排序（甚至能做外部排序）。
  • 缺点：空间开销大，不是原地排序。

9. 基于堆的归并排序

9.1 背景

  • 归并排序：常规归并是用递归+双指针合并有序数组。
  • 堆归并：适合合并多个有序子序列，尤其在外部排序（大文件、内存不够时）很常见。 👉 这里借助 小根堆，始终取最小的元素放入结果序列。

9.2 思路

假设我们有 k 个有序数组/切片：

  1. 每个数组取第一个元素，放入小根堆，堆元素保存 (值, 属于哪个数组, 下标)。
  2. 不断从堆顶取出最小值，放入结果。
  3. 从该值所属数组，取下一个元素继续放堆。
  4. 重复直到堆空。

👉 这样避免了复杂的递归合并，逻辑更直观。

9.3 关键点：Go 的堆

Go 内置了 container/heap，你需要实现一个最小堆：

type Item struct {
    value int   // 元素值
    arrID int   // 属于哪个数组
    idx   int   // 在数组中的位置
}

type MinHeap []Item

func (h MinHeap) Len() int            { return len(h) }
func (h MinHeap) Less(i, j int) bool  { return h[i].value < h[j].value }
func (h MinHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }

func (h *MinHeap) Push(x any) { 
    *h = append(*h, x.(Item)) 
}
func (h *MinHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

9.4 归并实现

func mergeKSortedArrays(arrs [][]int) []int {
    h := &MinHeap{}
    heap.Init(h)

    // 初始化：每个数组第一个元素入堆
    for i, arr := range arrs {
        if len(arr) > 0 {
            heap.Push(h, Item{arr[0], i, 0})
        }
    }

    var result []int

    for h.Len() > 0 {
        // 取最小值
        item := heap.Pop(h).(Item)
        result = append(result, item.value)

        // 来自哪个数组，就推进下一个
        if item.idx+1 < len(arrs[item.arrID]) {
            next := arrs[item.arrID][item.idx+1]
            heap.Push(h, Item{next, item.arrID, item.idx+1})
        }
    }
    return result
}

9.6 时间复杂度分析

  • 堆大小：k（数组个数）
  • 每次 Pop/Push：O(log k)
  • 总元素数：n
  • 整体复杂度：O(n log k)

👉 比两两合并的 O(k*n) 高效很多。

9.7 总结

  • 基于堆的归并适合多路合并。

  • 用 Go 的 container/heap 简单实现。

  • 思想就是：

    维护一个小根堆 → 每次取最小值 → 推入结果 → 补充该数组的下一个元素。
