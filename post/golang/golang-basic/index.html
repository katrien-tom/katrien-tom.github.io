1. Slice

1.1 Go 中切片的值语义和指针传递的场景

Go 切片（[]T）是一个结构体，包含三个字段：

type slice struct {
    ptr *T   // 指向底层数组的指针
    len int  // 当前长度
    cap int  // 容量
}

所以 切片本身是一个“小结构体”，通常 24 字节（在 64 位机器上）。

1.1.2 切片传递时的行为

  • 传切片（[]T）是 值传递，会复制这个 24 字节的结构体。
  • 复制的是切片的 描述信息（ptr、len、cap），不是复制底层数组。
  • 因此：
    • 在函数内修改切片元素（s[i] = ...）会影响外部（因为指向同一个底层数组）。
    • 但在函数内 改变切片的 len/cap 或重新切片，外部切片不会受影响（因为修改的是复制品的字段）。

1.1.3 什么时候直接传切片（[]T）

大多数场景都直接传切片，因为：

  • 切片描述符很小（24 字节），复制开销低。
  • 能直接修改底层数组的内容。
  • Go 的标准库大量 API 都是直接用切片，比如 append, bytes.Buffer, strings.Split 等。

适用情况：

  1. 只需要修改切片元素值，不需要改切片本身的长度/容量

    func fillZeros(s []int) {
    for i := range s {
        s[i] = 0 // 改元素
    }
}

  2. 读切片数据（不改长度、不扩容）

    func sum(s []int) int {
    total := 0
    for _, v := range s {
        total += v
    }
    return total
}

1.1.4 什么时候传切片指针（*[]T）

传指针的意义是： 让调用方看到你对切片本身（len、cap、底层数组引用）的改变。

适用情况：

  1. 需要修改切片的长度/容量，并让调用方看到

    func appendInPlace(s *[]int, vals ...int) {
    *s = append(*s, vals...) // 改的是调用方的切片变量
}

  2. 需要替换底层数组

    func reset(s *[]int) {
    *s = make([]int, 0) // 新切片，调用方变量指向它
}

  3. 切片结构本身很大（非常少见） 如果切片很大（比如切片是 [10^7]int），复制 24 字节没事，但复制切片结构体只是微不足道的成本，通常不会为了节省这点而用 *[]T。

  4. 避免返回值

    • 如果你想直接改传入的切片变量，而不是返回新切片，可以用 *[]T（不过 Go 更推荐返回新切片）。
    func filterInPlace(s *[]int) {
    result := (*s)[:0]
    for _, v := range *s {
        if v > 0 {
            result = append(result, v)
        }
    }
    *s = result
}

1.1.5 推荐习惯

  • 能传切片就传切片：简洁、惯用。
  • 确实需要改切片本身（len/cap/底层数组引用）才传指针。
  • 如果只是为了性能，通常没必要用 *[]T，除非你明确测过瓶颈。

1.3 切片内存共享和内存分离

​ 在 Go 中，切片（slice）是对数组的一种轻量级封装，它有时会涉及到内存共享和内存分离的问题，具体情况取决于切片的操作。理解切片内存的共享和分离可以帮助我们更好地优化代码和避免潜在的内存错误。

1.3.1 切片（Slice）的结构

​ 在 Go 中，切片并不直接持有数据，它是一个对底层数组的引用。一个切片的结构大致如下：

type Slice struct {
    ptr   *T  // 指向数组的指针
    len   int // 切片的长度
    cap   int // 切片的容量
}
  • ptr: 指向底层数组的指针。
  • len: 切片的元素个数。
  • cap: 底层数组从切片的起始位置开始可用的元素个数。

1.3.2 内存共享

​ 内存共享指的是多个切片共享底层的同一个数组。即使你创建了不同的切片，它们也可能指向同一个底层数组。当切片是从同一个数组派生出来时，修改其中一个切片的内容，会影响到其他切片的内容，因为它们指向同一个内存区域。

示例：内存共享

package main

import "fmt"

func main() {
    arr := []int{1, 2, 3, 4, 5}
    slice1 := arr[1:4] // 包含 arr[1], arr[2], arr[3]
    slice2 := arr[2:5] // 包含 arr[2], arr[3], arr[4]

    // 修改 slice1 的元素
    slice1[0] = 99

    // 输出 arr, slice1 和 slice2
    fmt.Println("arr:", arr)      // arr: [1 99 3 4 5]
    fmt.Println("slice1:", slice1) // slice1: [99 3 4]
    fmt.Println("slice2:", slice2) // slice2: [3 4 5]
}

在这个例子中，slice1 和 slice2 都共享同一个底层数组（arr），因此对 slice1 的修改会影响到 arr 和 slice2。

1.3.3 内存分离

​ 内存分离指的是当我们修改切片的容量（通过重新分配内存或使用 append 操作等），切片会创建一个新的底层数组，这时就不会再共享原数组的内存空间。

示例：内存分离

go复制编辑package main

import "fmt"

func main() {
    arr := []int{1, 2, 3, 4, 5}
    slice1 := arr[1:4] // 包含 arr[1], arr[2], arr[3]

    // 使用 append 导致内存分离
    slice2 := append(slice1, 6)

    // 输出 arr, slice1 和 slice2
    fmt.Println("arr:", arr)      // arr: [1 2 3 4 5]
    fmt.Println("slice1:", slice1) // slice1: [2 3 4]
    fmt.Println("slice2:", slice2) // slice2: [2 3 4 6]
}

在上面的代码中，append(slice1, 6) 创建了一个新的底层数组，因为原来 slice1 的容量已经达到了底层数组的限制。由于底层数组被重新分配，slice2 不再共享原始数组的内存，修改 slice2 不会影响到 arr 或 slice1。

1.3.4 内存共享和内存分离的总结

  • 内存共享：
    • 发生在切片从同一个底层数组派生时。
    • 一个切片的修改会影响到所有共享底层数组的切片。
    • 操作如 slice[i:j] 或者 append 不会改变原切片的容量时，底层数组会共享。
  • 内存分离：
    • 发生在使用 append 或其他操作导致切片的容量超出原数组时。
    • 此时，Go 会为切片分配新的底层数组，原数组和新的数组不再共享内存。
    • 通过 append 操作，如果容量不够，Go 会分配一个新的底层数组来存储数据。

1.3.5 性能和优化

  • 在进行切片操作时，需要注意内存共享和内存分离的性能差异。
    • 内存共享可以减少内存分配和复制的开销。
    • 内存分离则可能导致新的内存分配，因此要尽量避免不必要的 append 操作，尤其是在已知容量足够的情况下。

package main

import (
	"fmt"
	"time"
)

func main() {
	basicDefer()
	defer_exe_time()
	timeOfWork(1)
}
func basicDefer() {
	// defer典型的应用场景是释放资源,比如关闭文件句柄,释放数据库连接等
	fmt.Println("A")
	defer fmt.Println(1) //defer用于注册一个延迟调用（在函数返回之前调用）
	fmt.Println("B")
	defer fmt.Println(2) //如果同一个函数里有多个defer，则后注册的先执行
	fmt.Println("C")
	defer fmt.Println(3)
	fmt.Println("D")
}
func defer_exe_time() (i int) {
	i = 9
	defer func() { //defer后可以跟一个func
		fmt.Printf("first i=%d\n", i) //打印5，而非9。充分理解“defer在函数返回前执行”的含义，不是在“return语句前执行defer”
	}()
	defer func(i int) {
		fmt.Printf("second i=%d\n", i) //打印9。在注册defer时i就传进去了
	}(i)
	defer fmt.Printf("third i=%d\n", i) //defer后不是跟func，而直接跟一条执行语句，则相关变量在注册defer时被拷贝或计算
	return 5
}
func timeOfWork(arg int) int {
	begin := time.Now()
	defer func() { fmt.Printf("use time %f seconds\n", time.Since(begin).Seconds()) }()
	if arg > 10 {
		time.Sleep(2 * time.Second)
		return 100
	} else {
		time.Sleep(3 * time.Second)
		return 200
	}
}

7. Panic和Recover

recover()函数必须放在defer func(){}里才生效

package main

import "fmt"

// 何时会发生panic:
// 1. index out of range和divide by zero是常见的runtime error, 会发生panic
// 2. 通常系统初始化发生重大问题会主动调用panic(any)
// panic时会依次执行:
// 1. 执行已经注册的defer(后注册的先执行),未注册的defer不执行
// 2. 打印错误调用堆栈
// 3. 调用os.Exit(2)结束go进程

func defer_panic() {
	defer fmt.Println(1)
	var arr []int
	n := 0
	// defer fmt.Println(1 / n) // 在注册defer时就要计算1/n, 发生panic
	defer func() {
		_ = arr[n]
		_ = 1 / n            // defer func 内部发生panic, main协程不会exit, 其他defer还可以正常执行
		defer fmt.Println(2) // 上面那行代码发生发panic, 所以本行的defer没有注册成功
	}()
	defer fmt.Println(3)
}

// recover()一定要放在函数的最开始位置
func soo() {
	fmt.Println("enter soo")
	defer func() { //去掉这个defer试试,看看panic的流程。把这个defer放到soo函数末尾试试。把这个defer移到main()里试试。
		// recover必须在defer中才能生效
		if panicInfo := recover(); panicInfo != nil {
			fmt.Printf("soo函数中发生了panic:%v\n", panicInfo)
			// debug.PrintStack() //打印调用堆栈
		}
	}()

	fmt.Println("regist recover")

	defer fmt.Println("hello")
	defer func() {
		n := 0
		_ = 3 / n // 除θ异常,发生panic,下一行的defer没有注册成功
		defer fmt.Println("how areyou")
	}()
}

func main() {
	// defer_panic()
	soo()
}

8. Channel

代码

package main

import "fmt"

func rangeChannel() {
	var ch chan int //声明
	if ch == nil {
		fmt.Printf("ch is nil,ch len %d cap %d\n", len(ch), cap(ch))
	}
	// ch <- 2 //不能向nil chan里发送数据
	if len(ch) == 0 { //引用类型未初始化时都是nil,可以对它们执行len()函数,返回0
		fmt.Println("ch lengthis 0")
	}
	ch = make(chan int, 8) //初始化,环形队列里可容纳8个int
	ch <- 1                //往管道里写入(send)数据
	ch <- 2
	ch <- 3
	ch <- 4
	ch <- 5
	fmt.Printf("ch len %d cap %d\n", len(ch), cap(ch))
	v := <-ch //从管道里取走(recv)数据
	fmt.Println(v)
	v = <-ch
	fmt.Println(v)
	fmt.Println()

	close(ch)
	//遍历并取走（receive）管道里的元素。当管道里已无剩余元素且没有close管道时，receive操作会一直阻塞，最终报deadlock。当管道为空且被close后，for循环退出。
	for ele := range ch {
		fmt.Println(ele)
	}
	c := make(chan int, 10)
	send(c)
	recv(c)

}

// 只能向channel里写数据
func send(c chan<- int) {
	c <- 1
}

// 只能向channel里读数据
func recv(c <-chan int) {
	v := <-c
	fmt.Printf("take %d from read-only channel\n", v)
}
func main() {
	rangeChannel()
}

9. 引用类型介绍

slice、map、channel是go语言里的三大引用类型，如果只是想改变它们引用的底层数据，不需要传指针，因为传引用类型本质上传的就是底层数据的指针。

代码

package main

import (
	"fmt"
	"strings"
)

func changeArray(arr [3]int) {
	arr[0]++
}
func changeSlice(slc []int) {
	if len(slc) > 0 {
		slc[0]++
	}
}
func changeMap(mp map[int]bool) {
	mp[0] = true
}
func changeChan(ch chan bool) {
	if cap(ch) > len(ch) {
		ch <- true
	}
}
func main() {
	fmt.Println(strings.Repeat("*", 50)) //星号重复50次
	arr := [3]int{}
	changeArray(arr)
	fmt.Println(arr[0])
	slc := []int{1, 2, 3}
	changeSlice(slc)
	fmt.Println(slc[0])

	mp := map[int]bool{0: false}
	changeMap(mp)
	fmt.Println(mp[0])
	ch := make(chan bool, 3)
	changeChan(ch)
	fmt.Println(<-ch)
}

package main

func SumI(args ...EmptyInterface) int {
	rect := 0

		switch v := ele.(type) { // .(type)只能用在Switch后面

			fmt.Printf("不支持的数据类型 %[1]T 值为%[1]v\n", ele)
		}
	}
	return rect
}

	a = Car{}

	mmap := make(map[interface{}]interface{}, 10)
	mmap["a"] = 1
	mmap["b"] = "A"
	mmap["b"] = "A"

13.3 结构体方法接收值和接收指针的区别

13.3.1 方法接收者的两种形式

值接收者 (T)

func (t T) Method() {}
  • 调用时，接收者会 复制 一份。
  • 适合小对象、不需要修改原值。

指针接收者 (*T)

func (t *T) Method() {}
  • 调用时，传的是指针。
  • 能修改原值，避免大对象复制。

13.3.2 接口实现规则（重点！）

假设接口：

type I interface {
    M()
}

Go 有一条很重要的规则：

  1. 值接收者实现接口 → 值类型和指针类型都实现接口 因为指针可以自动解引用调用值方法。
  2. 指针接收者实现接口 → 只有指针类型实现接口 因为值调用指针方法时，需要获取值的地址，但 Go 不会在接口赋值时自动取地址。

13.3.3 示例对比

package main

import "fmt"

type MyStruct struct{}

type I interface {
    M()
}

// 值接收者
func (m MyStruct) M() {
    fmt.Println("value receiver")
}

func main() {
    var i I

    v := MyStruct{}
    p := &MyStruct{}

    i = v // ✅ OK
    i.M()

    i = p // ✅ OK
    i.M()
}

如果换成指针接收者：

type MyStruct struct{}

type I interface {
    M()
}

// 指针接收者
func (m *MyStruct) M() {
    fmt.Println("pointer receiver")
}

func main() {
    var i I

    v := MyStruct{}
    p := &MyStruct{}

    // i = v // ❌ 编译错误：MyStruct 没有实现 I
    i = p // ✅ OK
    i.M()
}

13.3.4 口诀记忆

  • 值接收者：值、指针都能用（更灵活，但会复制）。
  • 指针接收者：必须用指针（能改原值，也避免拷贝）。
  • 如果方法需要修改接收者内容，必须用指针接收者。
  • 如果实现接口需要用指针接收者，那么用值赋接口变量会编译报错。

13.3.5 小坑提醒

  • 如果结构体方法中包含 大字段（如大数组、切片、map），用值接收者可能会有性能问题，因为会复制。
  • 如果结构体是不可变的（纯数据承载，不会修改自身），用值接收者更直观。

14. Error

为什么需要 errors.Is / errors.As

Go 里很多时候我们会包装（wrap）错误，比如用 fmt.Errorf("xxx: %w", err) 或第三方库来加上下文信息。 这样做方便排查，但会带来一个问题：

  • 普通比较 err == target 会失效，因为包装后的 err 已经不是原来的对象了。
  • 我们需要一种方法，去 追踪错误链 找到它的“根错误”或特定类型。

这就是 errors.Is 和 errors.As 的作用。

14.1 errors.Is — 判断错误是否等于某个已知错误

作用

  • 顺着错误链查找，看里面有没有和 target 一样的错误。

签名

func Is(err, target error) bool

示例

import (
    "errors"
    "fmt"
)

var ErrNotFound = errors.New("not found")

func findSomething() error {
    return fmt.Errorf("query failed: %w", ErrNotFound)
}

func main() {
    err := findSomething()

    // 普通比较（false，因为包装过）
    fmt.Println(err == ErrNotFound) // false

    // errors.Is 会沿着链查找（true）
    fmt.Println(errors.Is(err, ErrNotFound)) // true
}

口诀

  • errors.Is：判断 是不是某个已知错误值，会一直顺着包装链去找。

14.2 errors.As — 判断错误是否是某种类型，并取出它

作用

  • 顺着错误链查找，是否有某个类型的错误，如果有，就把它赋值给你的变量。

签名

func As(err error, target interface{}) bool

注意：target 必须是 指向某个错误类型的指针。

示例

import (
    "errors"
    "fmt"
    "os"
)

func openFile() error {
    _, err := os.Open("no_such_file.txt")
    return fmt.Errorf("failed to open: %w", err)
}

func main() {
    err := openFile()

    var pathErr *os.PathError
    if errors.As(err, &pathErr) {
        fmt.Println("操作路径:", pathErr.Path)
        fmt.Println("原始错误:", pathErr.Err)
    } else {
        fmt.Println("不是 PathError")
    }
}

口诀

  • errors.As：判断 是不是某个类型的错误，还能拿到它的具体字段。

14.3 String()和Error()

14.3.1 背景

在 Go 里：

  • String() 是 fmt.Stringer 接口的方法

    type Stringer interface {
    String() string
}

    作用：告诉 fmt 系列函数（fmt.Println, fmt.Sprintf, etc.）如何把你的对象转成字符串。

  • Error() 是 error 接口的方法

    type error interface {
    Error() string
}

    作用：实现了 Error() 的类型就是一个 error，可以被当作错误使用。

14.3.2 如果一个类型同时实现了 String() 和 Error()

假设：

package main

import "fmt"

type MyType struct {
	msg string
}

func (m MyType) String() string {
	return "String(): " + m.msg
}

func (m MyType) Error() string {
	return "Error(): " + m.msg
}

func main() 
	m := MyType{"hello"}

	fmt.Println(m)  // Error(): hello
	fmt.Println(&m) // Error(): hello
	var err error = m
	fmt.Println(err) // Error(): hello
}

14.3.3 规则（重点）

普通变量

当 fmt 系列函数输出一个值时，会检查：

  1. 如果值是 error，直接调用它的 Error() 方法。
  2. 如果值实现了 fmt.Stringer，调用它的 String() 方法。
  3. 否则使用默认格式化规则。

例子解析:

用上面的代码：

fmt.Println(m)
  • m 类型是 MyType，虽然它实现了 Error()，但它并不是 error 类型的变量。
  • fmt 会优先找 String() → 输出 String(): hello
var err error = m
fmt.Println(err)
  • err 是 error 接口类型变量
  • 接口调用时，fmt 会先调用 Error() → 输出 Error(): hello

14.3.4 结论口诀

  • 普通值打印：fmt 会优先用 String()（如果实现了）。
  • error 类型变量打印：fmt 会用 Error()。
  • Error() 决定了错误信息，String() 决定了非错误状态下的字符串表现。

14.3.5 额外提示

  • 如果一个类型既实现 String() 又实现 Error()，那么它作为错误返回时会显示 Error() 的内容；作为普通结构体打印时会显示 String() 的内容。
  • 在调试和日志里，可能会因为接口类型转换导致输出不同，需要特别注意。

15. 闭包

15.1 什么是闭包

简单来说：

闭包（Closure）就是一个“带着它作用域里的变量一起打包出去”的函数。

在 Go 里，函数是一等公民，可以：

  • 作为变量存储
  • 作为参数传递
  • 作为返回值返回

当一个函数引用了它外部作用域中的变量，即使这个外部作用域已经结束，这些变量仍然会和函数一起活着，这就是闭包。

15.2 基本示例

package main

import "fmt"

func adder() func(int) int {
    sum := 0 // 被引用的外部变量
    return func(x int) int {
        sum += x
        return sum
    }
}

func main() {
    pos := adder()
    fmt.Println(pos(1)) // 1
    fmt.Println(pos(2)) // 3
    fmt.Println(pos(3)) // 6

    neg := adder()
    fmt.Println(neg(-1)) // -1
}

关键点

  • sum 在 adder() 返回后依然存在，因为它被闭包引用。
  • pos 和 neg 各有自己的 sum 副本，互不干扰。

15.3 闭包的特性

  1. 状态保存 外部变量在多次调用之间能记住之前的值。
  2. 生命周期延长 即使外部函数结束，变量依然存活。
  3. 引用捕获 捕获的是变量的引用，而不是值（这也是常见坑的来源）。

15.4 常见陷阱

在 Go 1.21 及以前：

  • for i := 0; i < 3; i++ { ... } 中的 i 是整个循环共享同一个变量
  • 闭包捕获的是这个变量的地址 → 所以执行时都会看到最后一次循环的值（3）

在 Go 1.22（release notes: Go 1.22 Release Notes - Loops）里改了规则：

  • 每次循环迭代都会生成新的 i 变量
  • 闭包捕获的是当次循环的那个变量
  • 所以你看到的地址每次都不一样，执行结果自然是 0、1、2

15.4.1 for 循环变量捕获问题

Go 1.21 及以前

func main() {
    var funcs []func()
    for i := 0; i < 3; i++ {
        funcs = append(funcs, func() {
            fmt.Println(i) // 捕获的是同一个 i
        })
    }
    for _, f := range funcs {
        f() // 输出 3 3 3
    }
}

原因

  • 在旧版本中，循环变量 i 在整个循环中是同一个变量。
  • 闭包捕获的都是它的引用，执行时取到的是最后的值。

修复方法（兼容旧版本 Go）

for i := 0; i < 3; i++ {
    v := i // 每轮循环复制一份变量
    funcs = append(funcs, func() {
        fmt.Println(v)
    })
}

Go 1.22 及以后

func main() {
    var funcs []func()
    for i := 0; i < 3; i++ {
        fmt.Printf("in for loop, address of i is %p\n", &i) // 每次输出的指针地址都不一样
        funcs = append(funcs, func() {
            fmt.Printf("in closure func, address of i is %p\n", &i)
            fmt.Println(i) // 注意：捕获的是同一个 i
        })
    }
    for _, f := range funcs {
        f() // 正常输出 0 1 2
    }
}

变化

  • 从 Go 1.22 起，for 和 range 循环的迭代变量每轮都会生成一个新的变量。
  • 闭包捕获的是不同的变量，避免了“全是最后一个值”的坑。

建议

  • 只用 Go 1.22+ → 直接写即可。
  • 需要兼容旧版本 → 保留 v := i 变量复制的写法。

15.5 使用场景

场景 1：工厂模式（生成带状态的函数）

func counter(start int) func() int {
    count := start
    return func() int {
        count++
        return count
    }
}

func main() {
    next := counter(10)
    fmt.Println(next()) // 11
    fmt.Println(next()) // 12
}

优点：不用全局变量，也不需要复杂的结构体。

场景 2：延迟执行 + 状态保持（缓存）

func fib() func() int {
    a, b := 0, 1
    return func() int {
        a, b = b, a+b
        return a
    }
}

func main() {
    f := fib()
    fmt.Println(f()) // 1
    fmt.Println(f()) // 1
    fmt.Println(f()) // 2
    fmt.Println(f()) // 3
}

优点：递归逻辑可以转成迭代，避免栈消耗。

场景 3：资源管理

func withFile(path string) func() {
    fmt.Println("打开文件:", path)
    return func() {
        fmt.Println("关闭文件:", path)
    }
}

func main() {
    closeFile := withFile("test.txt")
    // 处理文件
    closeFile() // 延迟关闭
}

场景 4：函数柯里化

func multiply(x int) func(int) int {
    return func(y int) int {
        return x * y
    }
}

func main() {
    times2 := multiply(2)
    fmt.Println(times2(5)) // 10
}

15.6 口诀记忆

  • 闭包就是函数 + 它引用的外部变量
  • 变量生命周期由引用决定，不由作用域决定
  • 循环里小心变量捕获坑
  • 工厂模式、缓存、资源管理、柯里化 都是闭包的常用玩法

16. 泛型

16.1 泛型基础回顾

泛型是指函数或类型能够操作任意类型的能力，避免代码重复，提高复用性。

Go 的泛型基于 类型参数（type parameters） 和 约束（constraints） 实现。

16.1.1 泛型函数示例

func Swap[T any](a, b T) (T, T) {
    return b, a
}

func main() {
    x, y := Swap[int](1, 2)           // 指定类型参数
    s1, s2 := Swap("hello", "world") // 类型推断
    fmt.Println(x, y, s1, s2)         // 2 1 hello world
}

16.1.2 泛型类型示例

type List[T any] struct {
    items []T
}

func (l *List[T]) Add(item T) {
    l.items = append(l.items, item)
}

16.2 类型参数约束（constraints）

约束用于限制泛型函数/类型支持的类型，any 是内置的无限制约束。

16.2.1 预定义约束示例

func PrintSlice[T comparable](s []T) {
    for _, v := range s {
        fmt.Println(v)
    }
}

comparable 表示类型支持 == 和 != 操作。

16.3 Go 1.24 新增特性：泛型类型别名

在 Go 1.24 之前，类型别名不能带类型参数。1.24 允许这样写：

package geometry

type Point[T any] struct {
    X, Y T
}

type MyPoint[T any] = Point[T] // 泛型类型别名

这样可以方便重构代码和跨包兼容，提升泛型代码的可维护性。

16.4 泛型编程常用模式

16.4.1 基于泛型和Map实现Set

用 map[T]struct{} 封装泛型 Set：

type Set[T comparable] struct {
    m map[T]struct{}
}

func NewSet[T comparable](items ...T) *Set[T] {
    s := &Set[T]{m: make(map[T]struct{})}
    for _, item := range items {
        s.m[item] = struct{}{}
    }
    return s
}

func (s *Set[T]) Add(item T) {
    s.m[item] = struct{}{}
}

func (s *Set[T]) Remove(item T) {
    delete(s.m, item)
}

func (s *Set[T]) Contains(item T) bool {
    _, ok := s.m[item]
    return ok
}

16.4.2 泛型约束自定义示例

自定义约束实现只能支持数字类型：

package constraints

type Number interface {
    ~int | ~int64 | ~float64 | ~uint
}

func Add[T Number](a, b T) T {
    return a + b
}

16.5 泛型性能和注意事项

  • 泛型代码在编译时会针对每个具体类型实例化（monomorphization），生成对应的代码。
  • 过度泛型可能导致二进制膨胀，需合理设计。
  • 使用内置约束和自定义约束提高安全性。

17. 进制练习-Excel一共多少列

背景

  • Excel 的列名是用 字母表示的 26 进制。
  • 单个字母：A ~ Z → 1 ~ 26
  • 两个字母：AA ~ ZZ
  • 三个字母：AAA ~ XFD
  • Excel 的最后一列是 XFD。

思路

列号计算 = 把字母当成 26 进制数，从左到右依次展开。

例如：

XFD = X * 26² + F * 26¹ + D * 26⁰

其中：

  • A = 1, B = 2, …, Z = 26

代码实现

package main

import "fmt"

// Excel最后一列是XFD，计算总列数
func main() {
	// 基础：A=1, Z=26
	fmt.Printf("A=%d Z=%d\n", 'A', 'Z')
	base := 26

	// 计算 "XFD"
	total := 0
	total += int('X'-'A'+1) * base * base // X * 26²
	total += int('F'-'A'+1) * base       // F * 26¹
	total += int('D'-'A'+1)              // D * 26⁰

	fmt.Println("Excel列总数 =", total) // 16384
}

输出

A=65 Z=90
Excel列总数 = 16384
